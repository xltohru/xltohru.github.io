动态规划 (Dynamic Programming)

通常我们把动态规划叫作 DP。

定义和方法论

DP 是什么？

应用分治法时，出现需要重复求解的子问题，就可使用动态规划来优化。

动态规划是一种表格法，仅求解一次重复的子问题，将结果存入表格中，下次直接查表，从而避免了重复求解。

什么时候可以使用 DP？

根据定义，我们可以得出，能够使用 DP 求解的问题具备两个关键特征：

  • 最优子结构：主问题的最优解，是由子问题的最优解构成的。这就是状态转移方程的基础。(子问题必须相互独立)

  • 重叠子问题：拆解主问题的过程中，出现被反复计算小问题。

使用 DP 的几个关键步骤

  1. 刻画最优解的结构

  2. 递归定义最优解的值：写出递归方程

  3. 计算最优解的值 (并构造最优解)

注：若需要构造最优解的结构，一般需要用额外表格存储划分的细节。

几种经典的 DP 问题

1. 钢条切割 (一维 DP)

  • 问题：有一根长度为 nnn 的钢条，已知长度为 1,2,…,n1, 2, \dots, n1,2,…,n 的短钢条分别卖多少钱，求最大利润。

  • 刻画最优解结构：
    假设最佳切割位置为 iii，其中长为 iii 的短钢条将被直接卖掉，剩下的那段长度为 n−in-in−i 则作为子问题处理。

  • 状态转移方程：

rn=max⁡1≤i≤n(pi+rn−i)r_n = \max_{1 \le i \le n} (p_i + r_{n-i}) rn​=1≤i≤nmax​(pi​+rn−i​)

  • 以 n=4n = 4n=4 为例，展示递归树
graph TD

  • 使用一个数组存储子问题的结果。给出实现思路：初始化子问题结果为一个不可能结果 (比如说负数)，来判断子问题是否被解决。

  • (可选) 构造最优解：使用一个辅助数组存储切割位置

2. 矩阵链乘法 (二维 DP)

  • 问题：计算矩阵连乘 A1A2…AnA_1 A_2 \dots A_nA1​A2​…An​。不同的加括号方式，会导致计算量差异。求最少的相乘次数。

  • 刻画最优解结构：对于 AiAi+1…AjA_i A_{i+1} \dots A_jAi​Ai+1​…Aj​，存在最优划分位置 kkk，将此问题划分为 AiAi+1…AkA_i A_{i+1} \dots A_kAi​Ai+1​…Ak​ 和 AkAk+1…AjA_k A_{k+1} \dots A_jAk​Ak+1​…Aj​ 两个子问题。

  • 状态转移方程：

m[i][j]=min⁡i≤k<j(m[i][k]+m[k+1][j]+pi−1pkpj)m[i][j] = \min_{i \le k < j} \big( m[i][k] + m[k+1][j] + p_{i-1} p_k p_j \big) m[i][j]=i≤k<jmin​(m[i][k]+m[k+1][j]+pi−1​pk​pj​)

  • 计算最优结果：
    需要两个二维数组：主表 m[i][j]m[i][j]m[i][j] 和辅助表 s[i][j]s[i][j]s[i][j]。
    • m[i][j]m[i][j]m[i][j] 记录计算区间 [i,j][i, j][i,j] 的最小代价。
  • (并构造最优解)：
    • s[i][j]s[i][j]s[i][j] 记录区间 [i,j][i, j][i,j] 的最优切分点 kkk。通过查找 sss 表，可以还原最优解的括号是怎么加的。

3. 最长公共子序列 (二维 DP)

  • 问题：求解最长公共子序列

注：子序列不要求连续，连续的叫作子串 (chap.32 字符串匹配则是研究子串的问题)

  • 刻画最优解结构：考察序列 XmX_mXm​, YnY_nYn​ 最后一个字符 (xmx_mxm​ 和 yny_nyn​)。若 xm=ynx_m = y_nxm​=yn​，就把这个字符加入到待求子序列中，子问题为 Xm−1X_{m-1}Xm−1​, Yn−1Y_{n-1}Yn−1​；若 xm≠ynx_m \ne y_nxm​=yn​，子问题为 XmX_mXm​, Yn−1Y_{n-1}Yn−1​ 和 Xm−1X_{m-1}Xm−1​, YnY_nYn​。

  • 状态转移方程：

c[i][j]={c[i−1][j−1]+1，xi=yjmax⁡(c[i−1][j],c[i][j−1])，xi≠yjc[i][j] = \begin{cases} c[i-1][j-1] + 1 & ， x_i = y_j \\ \max(c[i-1][j], c[i][j-1]) & ， x_i \neq y_j \end{cases} c[i][j]={c[i−1][j−1]+1max(c[i−1][j],c[i][j−1])​，xi​=yj​，xi​=yj​​

  • 计算最优结果：使用 c[i][j]c[i][j]c[i][j] 记录序列 XXX 的前 iii 个字符和序列 YYY 的前 jjj 个字符的 LCS 长度。

(可使用滚动数组，把空间从 O(mn)O(mn)O(mn) 压缩到 O(min(m,n))O(min(m,n))O(min(m,n)))

4. 最优二叉搜索树 (二维 DP)

  • 问题：构造一棵二叉搜索树，每个词搜索频率不同。安排节点位置，让“整体的期望搜索代价”最小。

  • 刻画最优解结构：在区间 [i,j][i, j][i,j] 内选择 kkk 为根节点。子问题为 [i,k−1][i, k - 1][i,k−1] 和 [k+1,j][k + 1, j][k+1,j]。注意：合并后，左右两棵子树里所有节点的深度增加了 1，则总代价等于左右树代价和，再加上 [i,j][i, j][i,j] 所有词的概率总和。

  • 状态转移方程：

e[i][j]=min⁡i≤k≤j(e[i][k−1]+e[k+1][j]+w(i,j))e[i][j] = \min_{i \le k \le j} \big( e[i][k-1] + e[k+1][j] + w(i, j) \big) e[i][j]=i≤k≤jmin​(e[i][k−1]+e[k+1][j]+w(i,j))

  • 计算最优结果：

    • 主表 e[i][j]e[i][j]e[i][j]：记录子树的最优期望代价。
    • 概率表 w[i][j]w[i][j]w[i][j]：记录区间概率之和，避免每次用 for 循环重复累加。

  • 构造最优解：

    • 辅助表 root[i][j]root[i][j]root[i][j]：记录所有根节点，可从上至下构造出这棵树。
