Chap.12 二叉搜索树 (Binary Search Trees)

1、基本结构

  • 每个节点最多有2个子节点
  • 节点大小顺序：left<self<rightleft < self < rightleft<self<right

2、遍历

每种遍历方法都有具体的应用背景

中序遍历 (Inorder Tree Walk)

先遍历左子树 → 打印根节点 → 遍历右子树

由于“左 < 根 < 右”的性质，中序遍历会按从小到大的顺序输出树中所有的关键字（树排序）。

注：快速排序的递归判定树，在结构上等同于构建的二叉搜索树。

前序遍历 (Preorder Tree Walk)

先打印根节点 → 左子树 → 右子树

通常用于复制一棵树（代价最低）。

后序遍历 (Postorder Tree Walk)

先左子树 → 右子树 → 打印根节点

通常用于安全地删除整棵树（由叶到根）。

3、查找与修改

二叉搜索树的基本操作（查找、插入、删除）的运行时间均为 O(h)O(h)O(h)，其中 hhh 为树的高度。

查找操作

  • 查找任意值（二分查找）：从根节点开始。目标值小则向左，目标值大则向右。
  • 查找最大值：疯狂地往右。
  • 查找最小值：疯狂地往左。
  • 查找前驱 (Predecessor)：先往左，再疯狂地往右。
  • 查找后继 (Successor)：先往右，再疯狂地往左。

修改操作

  • 插入：执行查找，直到遇到空节点并插入。
  • 删除：
    • 情况 1：无子节点，直接删除。
    • 情况 2：1 个子节点，用待删除节点的子节点替代它自己。
    • 情况 3：2 个子节点，考察后继节点的情况。
      1. 后继无子节点：直接用待删除节点的后继替代它自己。
      2. 后继有子节点（只会是右子节点）：先用待删除节点的后继替代它自己，再用后继节点的子节点替代后继节点。

4、随机构建的二叉搜索树 (Randomly Built BST)

随机构建的意思是：将关键字按随机顺序插入到一棵初始为空的树中，n!n!n! 种排列方式出现的概率相等。

待证明定理

包含 nnn 个不同关键字的随机构建二叉搜索树，其期望高度为 O(lg⁡n)O(\lg n)O(lgn)

定理与引理
